# SSH / SFTP 终端工具 · 规格（spec）

> 状态：草案 v1，待评审。归属：toolset 桌面应用新增工具「SSH 终端」。
> 配套文档：实施计划见 [`plan.md`](./plan.md)。

## 1. 背景与目标

在 toolset（Tauri v2 + Svelte 5）里新增一个 SSH 终端工具，覆盖日常登录运维：连接管理、交互式终端、文件传输。硬约束沿用全局诉求——**低内存、GNU 工具链可构建（不切 MSVC）、界面整洁**。

与现有工具的本质区别：SSH 会话是**有状态、异步、长连接**的，不能套用现有「core 纯函数 + 薄命令」的无状态模式（详见 plan）。

## 2. 范围（分期）

| 期 | 范围 | 验收即交付 |
|----|------|-----------|
| **P1** | 终端连通 + 连接管理（分组/克隆/导入导出/持久化）+ 主密码 vault + 标签页 + 小字体 + 快捷键 | 能连上主机、跑交互式 shell、管理多份连接、多标签 |
| **P2** | SFTP 图形文件传输 | 浏览远端目录、拖拽上传下载、进度 |
| **P3** | trzsz + lrzsz(rz/sz) 终端内传输 | 终端里 `rz`/`sz`、trzsz 可用 |

**明确不做（Out of scope）**：端口转发/隧道、SSH 跳板链、X11 转发、会话录制回放、云同步。后续可单列需求。

## 3. 名词

- **连接（Connection）**：一份保存的登录配置（主机、端口、用户、认证方式、分组、备注等）。
- **会话（Session）**：一次活动的 SSH 连接 + 一个 PTY/shell，对应 UI 上一个标签页。
- **Vault**：本地凭据保险箱，用主密码加密**密码类字段**。
- **密码类字段（secret fields）**：登录密码、私钥内容（PEM）、私钥口令。**仅这三类加密**，其余字段明文。

## 4. 功能需求

### FR-1 连接管理
- FR-1.1 新增 / 编辑 / 删除连接；字段见 §5 数据模型。
- FR-1.2 **分组**：连接可归入分组（单层即可），侧栏按分组树展示。
- FR-1.3 **克隆**：基于现有连接复制出一份新配置（密码类字段一并复制，需 vault 已解锁；未解锁则复制明文字段、密码留空）。
- FR-1.4 **导入 / 导出**：一键导出全部连接为单个文件；一键导入合并。导入跨机语义见 FR-2.5。
- FR-1.5 **本地持久化**：连接配置存于应用数据目录（非仓库内），重启保留。

### FR-2 凭据安全模型（主密码 vault）
- FR-2.1 **仅密码类字段加密**（密码、私钥 PEM、私钥口令）；主机/端口/用户/分组/备注等**明文**存储，任何时候可浏览、可导出、可管理。
- FR-2.2 **主密码无找回**：忘记主密码即无法解密已存密码类字段。提供「重置 vault」——清空所有密文字段、保留全部明文连接，重设新主密码后逐条重填密码即可恢复使用。
- FR-2.3 **惰性解锁**：浏览/编辑连接的明文部分**不需要**主密码；仅在「发起连接需要保存的密码」或「查看/编辑密码类字段」时，才弹一次主密码框解锁。进程内缓存解锁状态，可主动「锁定」。
- FR-2.4 首次使用任意密码类字段前需**设置主密码**（一次性）。
- FR-2.5 **导入跨机**：导出文件自带 KDF salt 与校验块，密码类字段为密文。导入时若提供该文件的主密码 → 解密并用当前 vault 主密码重新加密入库；不提供/不匹配 → 仅导入明文元数据，密码类字段留空并标记「需重填」。

### FR-3 终端
- FR-3.1 密码认证与私钥认证（私钥来自粘贴 PEM 或选本地文件；支持带口令私钥）。
- FR-3.2 交互式 PTY + shell；远端输出流式实时渲染，按键实时回传。
- FR-3.3 终端随面板尺寸变化做 resize（同步远端窗口大小）。
- FR-3.4 **首次连接主机指纹 TOFU**：首次记录主机公钥指纹，后续不一致时显著告警（不静默放行）。
- FR-3.5 连接失败/断开有明确状态提示；支持「重新登录」。

### FR-4 多开
- FR-4.1 **标签页**形态：单窗口多标签，每标签一个独立会话；新建、关闭、切换。
- FR-4.2 同一连接可开多个标签（多开同一主机）。

### FR-5 字体与快捷键
- FR-5.1 终端默认**小号等宽**字体，提供字号档位调节。
- FR-5.2 快捷键（默认，可改）：复制 `Ctrl+Shift+C`、粘贴 `Ctrl+Shift+V`（避开终端 `Ctrl+C` 中断）、新建标签 `Ctrl+Shift+T`、重新登录 `Ctrl+Shift+R`。

### FR-6 文件传输
- FR-6.1（P2）**SFTP**：复用同一 SSH 连接；远端目录浏览、上传/下载（拖拽）、传输进度。
- FR-6.2（P3）**lrzsz**：终端内 `rz`/`sz`，经 ZMODEM 协议桥接本地文件对话框（远端需装 lrzsz）。
- FR-6.3（P3）**trzsz**：trzsz 协议传输，大文件带进度（远端需装 trzsz）。trzsz 与 lrzsz 功能重叠，P3 内 **trzsz 优先**落地。

## 5. 数据模型

连接配置（持久化为 JSON）。**明文字段**与**密文字段**分离存储：

```jsonc
{
  "version": 1,
  "groups": ["生产", "测试"],
  "connections": [
    {
      "id": "uuid",
      "name": "web-01",
      "group": "生产",
      "host": "10.0.0.1",
      "port": 22,
      "username": "deploy",
      "auth": "password",            // password | key
      "keyPath": null,                // 私钥引用本地文件时存路径（明文）
      "note": "前端机",
      // —— 以下为密码类字段：始终密文，结构见 vault —— 
      "secret": {
        "password":   { "v": 1, "nonce": "b64", "ct": "b64" } | null,
        "keyPem":     { "v": 1, "nonce": "b64", "ct": "b64" } | null,
        "keyPass":    { "v": 1, "nonce": "b64", "ct": "b64" } | null
      }
    }
  ],
  "vault": {
    "kdf": { "algo": "argon2id", "salt": "b64", "mCost": 65536, "tCost": 3, "pCost": 1 },
    "verifier": { "nonce": "b64", "ct": "b64" }   // 加密固定明文，用于校验主密码
  },
  "knownHosts": { "10.0.0.1:22": "sha256-fingerprint" }
}
```

> 密文块的 AAD 绑定 `connectionId + ":" + 字段名`，防止把 A 连接的密码块挪用到 B。

## 6. 安全需求

- SEC-1 KDF：Argon2id（salt 随机生成；参数可调，默认 m=64MiB/t=3/p=1，解锁一次性开销，对低配机可下调到 ~19MiB）。
- SEC-2 字段加密：AES-256-GCM，每字段独立随机 12B nonce，AAD 如 §5。
- SEC-3 主密码校验：vault.verifier 解密成功即视为密码正确；失败提示「主密码错误」而非「数据损坏」。
- SEC-4 内存卫生：解锁派生出的 key 在锁定/退出时 zeroize；不把明文密码写日志。
- SEC-5 主机指纹 TOFU（FR-3.4）。
- **威胁模型与非目标**：vault 防的是「配置文件被拷走后离线读取密码」；**不**防本机已登录用户在 App 解锁后的内存读取，也**不**是完整密钥管理系统。明文元数据（主机/用户）本身不加密——这是 FR-2.1 的明确取舍。

## 7. 非功能需求

- NFR-1 **GNU 可构建**：所有新依赖须在 `x86_64-pc-windows-gnu` + `D:\minGw64` 下编译链接通过（P0 已验证 russh+ring+russh-sftp 可行），不引入需要 MSVC/NASM/clang 的依赖。
- NFR-2 **低内存**：会话与缓冲按需分配；空闲内存维持现有量级，多标签线性增长可控。
- NFR-3 **响应**：本地按键到回显无可感延迟；输出大量刷屏不卡死 UI（流式 + 背压）。

## 8. 验收标准

- **P1**：设主密码 → 新建带密码的连接 → 重启 App → 浏览连接清单（无需主密码）→ 连接时解锁一次 → 进入 shell 执行命令；开 ≥2 标签互不干扰；导出后在另一份数据目录导入并用同一主密码恢复密码；改主机指纹触发告警；忘记主密码可「重置 vault」保留明文连接。
- **P2**：SFTP 浏览远端目录、上传与下载各一个文件、看到进度、校验内容一致。
- **P3**：远端 `sz file` 触发下载、`rz` 触发上传；trzsz 同样可用。

## 9. 待澄清 / 风险

- RISK-1 P3 的 trzsz/lrzsz 需在 plan 阶段做 Rust 侧协议选型/选库（ZMODEM、trzsz 协议），可能需自实现，列为 P3 启动前的二次调研。
- CQ-1 私钥来源：仅支持「粘贴 PEM / 选文件」，是否需要 ssh-agent 集成？（默认不做，P1 之后再议）
- CQ-2 known_hosts 是否复用系统 `~/.ssh/known_hosts`，还是 App 自管一份？（默认 App 自管，避免污染系统文件）
